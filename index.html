Drones On Ice - Fall 2016

Created for my information security class, Drones on ice is an imaginary company delivering ice cream by drone. The iOS app lets users create an account, pick a flavor, pay with apple pay, then monitor their order progress. The associated web service, written in python with flask, keeps track of the orders and allows them to be viewed and fulfilled using an "employee website".

Order OSU - Spring 2016

He said no. The experience on mobile devices in the browser was subpar and adoption has been slow so I made an app that made it easy to order on an iPhone. The Game of Set - Spring 2016

Set is a card matching game. A "set" is 3 cards who for each of 4 properties (color, number, shape, and shading) are either all the same or all different. As part of my web applications class, my group was tasked with programming a version of the game in Ruby. The project had few requirements and was mostly open ended. My group and I decided to implement many optional features including a graphical interface, hints, multiplayer, scoring, and a timer. I'm very happy with the finished product. My group worked very well together and the game ended up being pretty cool in my opinon.

Price Checker - Spring 2015

The primary grocery vendor at the convenience store where I work updates their pricing every 2 weeks. These changes weren't always well communicated which left us with inconsistant margins, incorrect price tags, and items ringing up for the wrong amount. I wrote a C program to compare pricing from the vendor's SQLite database against our cash register's CSV item list and display inconsistencies. The program brought the pricing issue to the attention of the directors of dining who ultimately decided pricing should freeze during each semester.

Inventory - Fall 2014

Inventory is taken every Sunday in the convenience store where I work. It used to be that we would print an 18 page list of our 500 items from a spreadsheet then write in the counts of each product and type them back into the computer. This wasted a lot of time. I wrote a Java program that used a USB barcode scanner to make the process simpler. The user scans the item or its tag and types the number on the laptop. The program outputs a spreadsheet with all the data. I used the program every week for 2 years until dining services purchased inventory guns.
